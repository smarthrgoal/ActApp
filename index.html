خارطة طريق بناء تطبيق "نشاط" عبر MIT App Inventor

1. مراحل تنفيذ التطبيق

  1. تصميم الواجهات: بناء شاشات (التسجيل، الرمز، الصفحة الرئيسية) باستخدام Layouts لتنظيم الـ Tabs والـ Footer.
  2. إدارة البيانات: ربط الأماكن والمهتمين بقاعدة بيانات.
  3. المنطق البرمجي: استخدام اللبنات (Blocks) لبرمجة التفاعل بين الشاشات والبيانات.

2. مقارنة قواعد البيانات: TinyDB vs Firebase

وجه المقارنة TinyDB Firebase
طبيعة التخزين محلي (على هاتف المستخدم) سحابي (عبر الإنترنت)
مشاركة البيانات لا يمكن مشاركة البيانات بين المستخدمين تسمح بمشاركة البيانات لحظياً بين المستخدمين
الاستخدام الأمثل حفظ إعدادات بسيطة أو بيانات خاصة تطبيقات التواصل، الأماكن، والبيانات المشتركة

3. أمثلة برمجية

استخدام TinyDB (لحفظ اسم المستخدم محلياً):

call TinyDB.StoreValue(tag="userName", value="أحمد")

استخدام Firebase (لحفظ مكان جديد):

call FirebaseDB.StoreValue(tag="مكان_جديد", valueList=(اسم_المكان, الموقع))
